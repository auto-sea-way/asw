### Task 2: Update CHANGELOG.md

**Files:**
- Modify: `CHANGELOG.md`

- [ ] **Step 1: Add v0.3.1 section**

Insert between `## [Unreleased]` and `## [0.3.0]`:

```markdown
## [0.3.1] - 2026-03-24

### Fixed

- Nearest-node snapping regression: routes to remote islands (Grenada, Palagruza) and coastal towns (Gallipoli, Monopoli) now resolve correctly
- Coastal snap quality restored to v0.2.0 level — ports snap to nearby fine-resolution nodes instead of distant coarse ones
- Adaptive two-pass snapping: fast k=3 scan handles 99% of queries, proportional refinement only when needed

### Performance

- Short/medium routes 2-18x faster than v0.2.0 (H3 binary search + pre-allocated A* buffers)
- Panama Canal: 47x faster (51 nm through canal vs 10,340 nm around continent in v0.2.0)
```

### Task 5: Trigger release workflow

**Files:** None (GitHub Actions)

- [ ] **Step 1: Trigger the release**

```bash
gh workflow run release.yml -f version=0.3.1 -f graph_tag=v0.3.0
```

The `draft-graph` tag points to the existing graph file (unchanged for this serve-only fix).

- [ ] **Step 2: Monitor the release workflow**

Run: `gh run list --workflow=release.yml --limit 1`

Wait for completion. The workflow will:
1. Run CI checks
2. Cross-compile binaries (linux-amd64, linux-arm64, darwin-amd64, darwin-arm64)
3. Create annotated tag `v0.3.0`
4. Create GitHub release with binaries + graph + checksums
5. Build and push Docker images (`0.3.1`, `latest`, `0.3.1-full`, `latest-full`)

- [ ] **Step 3: Verify release artifacts**

```bash
gh release view v0.3.1
```

Expected: 6 assets (4 binaries + graph + SHA256SUMS), changelog body matches.
